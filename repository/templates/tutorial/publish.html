In the Submit page, Developers can submit a new VNF to be integrated in the FENDE catalog. To do so, Developers can fill the form available with the following information:

  • Institution: name of the institution which supports the VNF development;
  • VNF Name: name of the VNF to be displayed in the FENDE catalog;
  • Version: the version of the VNF (e.g., 1.5);
  • Category: the category that represents the VNF functionality;
  • Abstract: a brief description of the VNF. It will be displayed in the FENDE catalog;
  • Full Description: a complete description of the VNF;
  • Link: a link to the Git repository which hosts the VNF and his descriptors according to FENDE repository standards. See Developerâ€™s Documentation for more information.

In the Registration Requests page, the developer can access information about all VNFs submitted to the FENDE catalog and their respective status (i.e., Accepted, To Review, and Rejected). Once reviewed, comments from the reviewers can be accessed by clicking on "See Comments" button when available. For information organizing, two distinct tables are presented in the Registration Requests page, one for the requests for insert new VNFs in the catalog (Repository Requests) and other for the requests to update a VNF (new version) already registered in the catalog (Update Requests).
