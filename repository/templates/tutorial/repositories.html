In the My Repositories page, developers can access information and manage/update his/jer accepted repositories. All repositories listed are available for customers to be acquired in the FENDE catalog.

Navigate through the items below to obtain more information about each manage/update action.

1Change Policy 2Update VNFD 3New Version

Updates the policies related to the privacy of selected repository. The policies available includes:

  • Contract VNF: Defines who (specific customer or institution) can use the VNF offered in the catalog. The values for this input are None, All or Specific.
  • See code source: Sets who (specific customer or institutio) can have access to the link for the VNF source code, in order to have development access to the VNF. The values for this input are the same of the policy above.

Updates the VNF Descriptor (VNFD) related to the selected repository. The developer must fill the form with comments about the modifications performed to motivate the VNFD update request. A file with the new descriptor must be sent as well.

Sends a new version of a existing repository. This option is useful to manage distinct versions of VNFs and allows developers to offer new repository (i.e., VNFs) versions to fix bugs or improve functionalities of an specific VNF, for example. The version history will be available for a customer whose want to contract the VNF.
