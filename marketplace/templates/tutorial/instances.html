In the Instances page, you can manage all VNFs instantiated. Besides, you can see details about each instance available, such as IP address and VNF status (i.e., Running, Stopped or Paused). In the last column of the Instances table, you can use a set of options related to specific VNF instances.

Navigate through the items below to obtain information about each available action.

  • 1Restart
  • 2Stop
  • 3Delete
  • 4Update
  • 5Resize Allocation
  • 6Info
  • 7Statistics
  • 8Logs

Restarts the selected instance. This action can be useful to apply new settings or when the instance appears to be misbehaving.

Stops the selected instance. When stopped, the instance will not process packets neither compose service chains.

Removes the selected instance from the infrastructure. Once deleted, all customized settings of the VNF will be lost. A new instance with standard configurations can be created in the VNF page.

This option updates the descriptors of the selected VNF instance.

This action allows the customer to set a new amount of resources to be used for supply the customer demands (e.g., memory, CPUs, and disk). The maximum amount of resources available is defined according to the contracted plan.

Shows detailed information about the selected instance and respective VNF repository.

Shows real-time statistics about the selected instance. These statistics include the resources being used by the instance (i.e., CPU, memory, and disk) and the number of packets being processed(transmitted and received). It is useful for performance analysis and network management activities.

Shows relevant low-level operational information about the life cycle of the selected instance. It includes all records from the execution until the instance be removed from the infrastructure.
